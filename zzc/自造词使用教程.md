# 自造词使用教程

教程较长，建议先看“常用指令速查”，再按自己的设备阅读后面的导出与合并流程

## 使用前注意

- 自造词支持 3-6 码
- 自造词入口键为 `\`，大多数指令也用第二个 `\` 结束
- “编码”指词汇编码，例如 `bmms`
- 自造词状态下，可以正常用顶功、数字、空格选字
- `\自造词3/4/5/6` 或 `\自造词三/四/五/六` 会在输入码长后自动结束，不需要再输入第二个 `\`
- 手机皮肤如果用上划触发 `\`，实际显示或内部按键可能不同，但只要能进入自造词模式即可

## 常用指令速查

| 指令                      | 作用                             |
| ------------------------- | -------------------------------- |
| `编码\自造词\`            | 空码添加词汇，或替换当前编码首选 |
| `\自造词3` 或 `\自造词三` | 指定自造词为 3 码                |
| `\自造词4`或 `\自造词四`  | 指定自造词为 4 码                |
| `\自造词5`或 `\自造词五`  | 指定自造词为 5 码                |
| `\自造词6`或 `\自造词六`  | 指定自造词为 6 码                |
| `编码\+自造词\`           | 给当前编码追加重码候选           |
| `编码\-\`                 | 删除当前编码首选                 |
| `编码\-数字\`             | 删除当前编码第 N 个候选          |
| `编码\<\`                 | 把当前编码候选前移一码           |
| `\--\`或 `编码\--\`       | 撤回最近一次未合并操作           |
| `\!!!\`或 `编码\!!!\`     | 清空全部未合并自造词操作         |

## 指令说明

### 1. 添加或替换词汇

```text
编码\自造词\
```

用途：

1. 当前编码为空码时，直接添加词汇
2. 当前编码已有首选时，新词会替换首选
3. 原首选会自动顺延到更长编码

顺延示例：

- 4 码首选被替换后，原词会尝试顺延到 5 码
- 如果 5 码也有首选，会继续顺延到 6 码
- 如果 6 码也有候选，会变成 6 码重码候选

### 2. 指定码长造词

指定码长：

```text
\自造词3
\自造词四
```

说明：

- 码长只能是 `3/4/5/6` 或 `三/四/五/六`
- 指定码长后会自动提交，不需要再输入 `\`
- 如果目标码长已有词汇，原词仍会按规则顺延

### 3. 追加重码候选

```text
编码\+自造词\
```

用途：给当前编码添加重码候选

说明：

- 不限制重码数量
- 新词不会抢首选，会排在已有候选后面
- 电脑输入 `+` 时，在自造词指令下直接按 `=` 键即可，不需要按 `Shift+=`

### 4. 删除候选

```text
编码\-\
编码\-数字\
```

用途：

- `编码\-\`：删除当前编码首选
- `编码\-2\`：删除当前编码第 2 个候选

适合清理重码

### 5. 前移一码

```text
编码\<\
```

用途：把当前编码候选向前移动一码

示例：

- 5 码词汇前移后，会变成 4 码
- 原 4 码词汇会自动顺延

说明：

- 只能由长码前移到短码，不能向后移动
- 最短移动到 3 码
- 适合把常用但靠后的词汇提前
- 电脑使用时，进入自造词模式后直接按 `,` 逗号键即可，不需要按 `Shift+,`

### 6. 撤回最近一次操作

```text
\--\
编码\--\
```

用途：撤回最近一次未合并的自造词操作

适合操作失误或临时后悔

### 7. 清空全部未合并操作

```text
\!!!\
编码\!!!\
```

用途：清空重新部署或合并前的全部自造词操作

注意：

- 这是紧急指令，范围较大
- 电脑输入 `!` 时，需要按 `Shift+1`

## 自造词如何保存和合并

自造词不是立刻写进正式码表，而是分几步生效：

1. 使用自造词指令添加、删除、调整词汇
2. 重新部署后，自造词操作会整理到 `zzc.dict.yaml`
3. 执行 `zzc` 文件夹内的合并脚本
4. 再次重新部署后，正式码表永久生效

## 元书入门版配置

适合不使用 iCloud 联动的用户

1. 把 `include_keyboard_rime_files.txt` 和方案文件放在一起
2. 使用自造词后，在元书里重新部署
3. 重新部署会把自造词整理到 `zzc.dict.yaml`
4. 把整个方案文件夹拷贝至电脑，不要拷贝 `build` 文件夹
5. 在 `zzc` 文件夹内选择适合自己电脑系统的合并脚本，直接执行
6. 把合并后的相关文件替换回输入法目录
7. 再次重新部署，正式生效

## 元书进阶版配置：iCloud 联动

适合使用苹果 iCloud 同步手机和电脑的用户

### 初次配置

1. 准备这 3 个同步清单文件：
   - `include_keyboard_rime_files.txt`
   - `exclude_iCloud_rime_files.txt`
   - `include_iCloud_rime_files.txt`
2. 在元书输入法的 `RimeUserData` 文件夹下新建方案目录，例如：
   - `txjx`
   - `xmjd6`
3. 把方案文件和上面 3 个同步清单文件放入该方案目录
4. 把方案目录放入 iCloud
5. 等待 iCloud 同步完成

不要放入 `build` 文件夹

### 日常使用流程

1. 在手机上使用自造词指令添加词汇
2. 点击重新部署，把自造词操作保存到 `zzc.dict.yaml`
3. 打开元书右下角菜单，选择 iCloud 上传功能
4. 等待 `zzc.dict.yaml` 和 `zzc_state` 里的指定文件上传到 iCloud
5. 电脑打开 iCloud 文件夹，等待文件同步完成
6. 在电脑上执行 `zzc` 文件夹内的合并脚本
7. 等待 iCloud 再次同步完成
8. 回到元书，点击重新部署

完成后，之前的自造词操作会正式永久生效

### iCloud 同步注意

- 同步有延迟，建议观察文件修改时间
- 如果第一次重新部署没有生效，可以稍等后再重新部署
- 如果重新部署时间较长，或提示正在拷贝 iCloud 文件，通常说明文件已经更新
- 如果一直不成功，手动检查 iOS 文件 App 里的元书输入法 iCloud 文件夹，确认是否完成同步

## 重新部署和合并频率建议

- 自造词添加后会占用键盘内存
- 不需要每添加一次就重新部署
- 也不建议等到提醒上限再处理
- 当前提醒范围未执行重新部署前大致为 1000 条至 3000 条
- 重新部署后，合并前的 `zzc.dict.yaml` 目测最高可容纳约 20000 条
- 到 10000 条会产生一次提示，15000和20000分别提示出现次数增加
- 如果每天都有自造词数据，建议每月合并一两次

## 不使用 iCloud 的导出方式

如果不用苹果 iCloud：

1. 导出 `zzc_state` 文件夹
2. 导出方案词库文件
3. 导出 `zzc.dict.yaml`
4. 在电脑执行合并脚本
5. 把合并后的文件替换回输入法目录

合并后的文件导入并重新部署后，相关临时数据会自动清空，不需要手动清理

## 电脑用户使用步骤

1. 使用指令添加自造词
2. 重新部署，保存到 `zzc.dict.yaml`
3. 执行合并脚本
4. 再次重新部署，正式生效

不需要频繁执行完整合并流程

如果需要 iCloud 联动：

1. 电脑安装 iCloud
2. 使用天行键同步助手
3. 按参考设置配置同步助手
4. 同步助手会自动拷贝 iCloud 文件到本机 Rime 用户文件夹
5. 自造词相关文件不受影响
6. 之后仍按“添加自造词 -> 重新部署 -> 合并脚本 -> 再次重新部署”的流程使用

天行键同步助手下载链接（星猫键道可用）

天行键同步助手：https://github.com/wzxmer/folder-sync-mirror

## 仓输入法说明

仓输入法可以使用自造词，但同步方式和元书不同，建议按手动导出流程使用

使用前请注意：

- 不建议开启 iCloud 同步
- 不要在 Rime 设置中长期打开“部署时覆盖键盘词库文件”
- 需要把 `Hamster.yaml` 和方案文件放在同一目录
- `Hamster.yaml` 用来让仓识别自造词相关文件，避免重新部署时丢失手机端操作

日常流程：

1. 在手机上使用自造词指令添加词汇
2. 打开仓输入法文件管理
3. 在通用功能里执行“使用键盘文件覆盖应用文件”
4. 点击重新部署，把自造词操作整理到 `zzc.dict.yaml`
5. 打开 iOS 文件 App
6. 在“我的 iPhone”里找到 `Hamster/Rime` 目录
7. 导出 `dict` 目录和 `zzc_state` 文件夹
8. 在电脑上执行 `zzc` 文件夹内的合并脚本
9. 把合并后的文件替换回手机仓输入法对应目录
10. 回到仓输入法重新部署，正式生效

如果只是临时造词，完成第 4 步后通常已经可以继续使用；如果要把自造词正式合并进码表，再执行后面的导出和电脑合并流程

iOS 用户如果需要更稳定的 iCloud 联动，建议优先使用元书输入法

## 安卓及其他平台说明

目前没有安卓设备实测，只能提供原理参考

基本流程：

1. 重新部署前，自造词内容保存在 `zzc_state` 文件夹内的 TSV 文件中
2. 重新部署时，TSV 数据会被整理到 `zzc.dict.yaml`
3. 导出 `zzc.dict.yaml` 和 `zzc_state`
4. 执行合并脚本
5. 合并脚本会把 `zzc.dict.yaml` 的内容插入正式码表，并更新 `zzc_reset` 文件
6. 把合并后的自造词文件和 `zzc_reset` 文件导入输入法
7. 输入法识别 `zzc_reset` 后，会自动清理旧运行数据

如果不确定需要哪些文件，直接拷贝整个 `zzc_state` 文件夹更稳

## 支持与反馈

如果使用中遇到问题，建议反馈时附上：

1. 使用的输入法平台，例如元书、仓、小狼毫、鼠须管等
2. 使用的方案，例如 `txjx` 或 `xmjd6`
3. 具体输入的自造词指令
4. 问题发生前后的候选显示或文件同步情况

这样更容易判断问题出在指令、候选显示、重新部署、同步，还是合并脚本
